## 🎯 最佳实践

### 1. 连接池配置

根据负载调整连接池大小：
```bash
# 环境变量配置
export DB_POOL_MIN_SIZE=5          # 最小连接数
export DB_POOL_MAX_SIZE=20         # 最大连接数
export DB_POOL_MAX_IDLE_TIME=300   # 空闲超时（秒）
export DB_POOL_HEALTH_CHECK_INTERVAL=60  # 健康检查间隔（秒）
```

**推荐配置**:
- 小型应用: min=3, max=10
- 中型应用: min=5, max=20
- 大型应用: min=10, max=50

### 2. 分页策略选择

**传统分页** (offset/limit):
- ✅ 适合: 小数据集（<10000条）
- ✅ 优点: 实现简单，支持跳页
- ❌ 缺点: 深分页性能差

**游标分页** (cursor-based):
- ✅ 适合: 大数据集（>10000条）
- ✅ 优点: 性能稳定，不受数据量影响
- ❌ 缺点: 不支持跳页

### 3. 查询优化建议

**DO ✅**:
- 使用索引字段作为WHERE条件
- 限制返回字段（select指定列）
- 添加LIMIT限制结果数量
- 使用游标分页处理大数据集
- 批量操作代替循环查询

**DON'T ❌**:
- 避免SELECT *（除非需要所有字段）
- 避免深分页（offset > 1000）
- 避免在未索引字段上使用LIKE
- 避免N+1查询问题
- 避免在循环中执行数据库操作

### 4. 监控和告警

定期检查性能指标：

## 🔍 故障排查

### 连接池问题

**问题**: 连接池耗尽
```
Connection pool exhausted, waiting for available connection
```

**解决**:
1. 增加最大连接数: `DB_POOL_MAX_SIZE=30`
2. 检查是否有连接泄漏（未正确释放）
3. 优化慢查询，减少连接占用时间

### 慢查询问题

**问题**: 查询时间过长
```
Slow query detected: select on messages took 2.5s
```

**解决**:
1. 检查是否使用了索引字段
2. 添加LIMIT限制结果数量
3. 使用游标分页代替offset分页
4. 考虑添加新索引

### 索引创建失败

**问题**: 索引创建超时或失败

**解决**:
1. 使用CONCURRENTLY避免锁表
2. 在低峰期创建索引
3. 增加数据库maintenance_work_mem
4. 分批创建索引，不要一次全部创建
